üìå 4.2 Identify and Handle Events

üîπ What Are Events in JavaScript?

Events are actions that happen in the browser ‚Äî like a user clicking a button, pressing a key, or changing an input field. JavaScript lets you listen for these events and respond using event handlers like onclick, onkeyup, onload, etc.

‚úÖ Common Events with Code Examples

üñºÔ∏è 1. onload ‚Äì When the Page Loads 
<body onload="pageLoaded()">

<script>
  function pageLoaded() {
    alert("Page has fully loaded!");
  }
</script>

üîç How it Works:

  • The <body> tag includes the onload attribute: <body onload="pageLoaded()">
  • This event fires when the page and all its assets have fully loaded.
  • It calls the pageLoaded() function, which shows an alert.

üñ®Ô∏è Expected Output:

When the user opens or refreshes the page, an alert pops up saying:

Page has fully loaded!

üî• Alternative: Using window.onload (Recommended)

window.onload = function() {
  alert("Page has fully loaded!");
};

This method separates HTML and JavaScript ‚Äî a cleaner and more modern practice.

üñ±Ô∏è 2. onclick ‚Äì When an Element is Clicked 
<button onclick="sayHello()">Click Me</button>
  
  <script>
    function sayHello() {
      alert("Button was clicked!");
    }
  </script>

üîç How it Works:

  • Button Element: <button onclick="sayHello()">Click Me</button>
    • This creates a button labeled "Click Me".
    • The onclick attribute triggers sayHello() when clicked.
  • JavaScript Function: 
    function sayHello() {
    alert("Button was clicked!");
  }
    • When executed, it displays an alert box with the message: "Button was clicked!".

üñ®Ô∏è Expected Output:

When you click the button, an alert box pops up with the text:
Button was clicked!

üî• Alternative: Using addEventListener() (Recommended) 

<button id="myButton">Click Me</button>
  
  <script>
    document.getElementById("myButton").addEventListener("click", function() {
      alert("Button was clicked!");
    });
  </script>

‚úÖ This method keeps HTML and JavaScript separate for better code structure and readability.

üîë 3. onkeydown, onkeyup, onkeypress ‚Äì Keyboard Events 
<input type="text" onkeydown="keyDown()" onkeyup="keyUp()" onkeypress="keyPress()">
  
  <script>
    function keyDown() {
      console.log("Key is being pressed down");
    }
    function keyUp() {
      console.log("Key was released");
    }
    function keyPress() {
      console.log("Key was pressed");
    }
  </script>

üîç How it Works:

  • Input Field: <input type="text" onkeydown="keyDown()" onkeyup="keyUp()" onkeypress="keyPress()">
    • Listens for keyboard events while typing.
  • Keyboard Event Functions:
    • onkeydown="keyDown()" ‚Äì Triggers when a key is pressed down.
    • onkeyup="keyUp()" ‚Äì Triggers when a key is released.
    • onkeypress="keyPress()" ‚Äì Triggers when a key is pressed (deprecated in modern browsers).

üñ®Ô∏è Expected Output in Console:

If you type "A" inside the input box, you will see:
Key is being pressed down
Key was pressed
Key was released

üî• Better Approach: Using addEventListener()

<input type="text" id="textInput">
  
  <script>
    let inputField = document.getElementById("textInput");
    inputField.addEventListener("keydown", function() {
      console.log("Key is being pressed down");
    });
    inputField.addEventListener("keyup", function() {
      console.log("Key was released");
    });
    inputField.addEventListener("keypress", function() {
      console.log("Key was pressed");
    });
  </script>

‚úÖ This approach keeps HTML and JavaScript separate, improving maintainability and readability.

üìù 4. onchange ‚Äì Input or Select Value Changed 
<select onchange="optionChanged(this)">
        <option value="Banana">Apple</option>
        <option value="Grapes">Banana</option>
        <option value="Watermelon">Grapes</option>
        <option value="Apple">Watermelon</option>
      </select>
      
      <script>
        function optionChanged(selectElement) {
          alert("You selected: " + selectElement.value);
        }
      </script>

üîç Explanation:

  • <select onchange="optionChanged(this)">
    • This dropdown has 4 options: Apple, Banana, Grapes, Watermelon.
    • The onchange event fires when the user changes the selection.
    • this refers to the entire <select> element.
  • optionChanged(selectElement)
    • This function is called when the selection changes.
    • selectElement.value gives the selected option's value.

üñ®Ô∏è Example Output:

If the user selects Banana, this alert appears:
You selected: Banana

üî• Better Version (Using JavaScript Only):

<select id="fruitSelector">
        <option value="apple">Apple</option>
        <option value="banana">Banana</option>
      </select>
      
      <script>
        document.getElementById("fruitSelector")
          .addEventListener("change", function() {
            alert("You selected: " + this.value);
          });
      </script>

‚úÖ Separates HTML and JavaScript, improving code cleanliness and readability.

üß† 5. onfocus and onblur ‚Äì Focus In and Out of Input 
<input type="text" onfocus="gotFocus()" onblur="lostFocus()" 
        placeholder="Enter your name">
      
      <script>
        function gotFocus() {
          console.log("Input got focus");
        }
      
        function lostFocus() {
          console.log("Input lost focus");
        }
      </script>

üîç How it Works:

  • onfocus Event
    • Fires when the input gains focus (user clicks inside).
    • Calls gotFocus() ‚Üí Logs Input got focus.
  • onblur Event
    • Fires when the input loses focus (user clicks away).
    • Calls lostFocus() ‚Üí Logs Input lost focus.

üñ®Ô∏è Expected Console Output:

  • Click inside the input: Input got focus
  • Click outside the input: Input lost focus

üî• Better Approach (Using addEventListener):

<input type="text" id="nameInput" placeholder="Enter your name">
      
      <script>
        let inputField = document.getElementById("nameInput");
      
        inputField.addEventListener("focus", function() {
          console.log("Input got focus");
        });
      
        inputField.addEventListener("blur", function() {
          console.log("Input lost focus");
        });
      </script>

‚úÖ Advantages:

  • Keeps HTML and JS separate and cleaner.
  • Allows multiple event listeners without conflict.

üöÄ Highlight Input on Focus (Bonus):

inputField.addEventListener("focus", function() {
        inputField.style.backgroundColor = "lightyellow";
      });
      
      inputField.addEventListener("blur", function() {
        inputField.style.backgroundColor = "";
      });

üé® Now, the background turns yellow when focused and resets when blurred.

üñ±Ô∏è 6. onmouseover and onmouseout ‚Äì Mouse Hover and Exit 
<p onmouseover="hoverText(this)" onmouseout="normalText(this)">Hover over me!</p>
      
      <script>
        function hoverText(element) {
          element.style.color = "blue";
          element.innerText = "You're hovering!";
        }
      
        function normalText(element) {
          element.style.color = "black";
          element.innerText = "Hover over me!";
        }
      </script>

üîç How it Works:

  • onmouseover Event
    • Triggered when the mouse moves over the <p> element.
    • Calls hoverText(this) ‚Üí changes color to blue and updates text.
  • onmouseout Event
    • Triggered when the mouse leaves the element.
    • Calls normalText(this) ‚Üí resets color and text.

üñ®Ô∏è Expected Behavior:

  • üìå Before Hover: Hover over me! (Black color)
  • üîµ On Hover: You're hovering! (Blue color)
  • üìå On Mouse Out: Resets back to Hover over me! (Black color)

üî• Better Approach (Using addEventListener):

<p id="hoverParagraph">Hover over me!</p>
      
      <script>
        let paragraph = document.getElementById("hoverParagraph");
      
        paragraph.addEventListener("mouseover", function() {
          paragraph.style.color = "blue";
          paragraph.innerText = "You're hovering!";
        });
      
        paragraph.addEventListener("mouseout", function() {
          paragraph.style.color = "black";
          paragraph.innerText = "Hover over me!";
        });
      </script>

‚úÖ Advantages:

  • Separates HTML and JavaScript for cleaner code.
  • Supports multiple listeners without conflict.

üé® Bonus: Smooth Transition on Hover

p {
        transition: color 0.3s ease-in-out;
      }

‚ú® The color change happens smoothly when hovering over the element.
