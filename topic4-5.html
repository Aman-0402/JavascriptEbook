üîπ 1. HTML Attribute Event

‚úÖ Use: Attach an event directly in an HTML tag (though not the best practice, it's simple and quick).

‚úÖ JS Code:

  function greetUser() {
    alert("Hello from HTML attribute event!");
  }

‚úÖ Basic Usage

You need to link this function to an element in your HTML. Example:

<script>
  function greetUser() {
    alert("Hello from HTML attribute event!");
  }
</script>

üõ† How it Works:

  • The <button> has an onclick event.
  • When clicked, it calls the greetUser() function.
  • The function triggers an alert saying: 
    "Hello from HTML attribute event!"

üéØ Alternative Ways to Add Events

‚úÖ Advantage: Keeps HTML clean and allows multiple event listeners.

‚úÖ 3. addEventListener()

Recommended way. Allows multiple event listeners on the same element and clean separation of HTML & JS.

üîπ Syntax:

element.addEventListener(event, function);

üîπ Example:

<button id="listenBtn">Click using addEventListener</button>
<script>
  const btn = document.getElementById("listenBtn");
  btn.addEventListener("click", function () {
    alert("Hello from addEventListener!");
  });
</script>

üéØ Why addEventListener is Better?

‚úÖ Allows multiple event listeners

btn.addEventListener("click", function () {
  console.log("Another event listener added!");
});

Now, both the alert and console.log will execute!
This wouldn‚Äôt be possible with .onclick, which replaces previous handlers.

‚úÖ Supports multiple event types

btn.addEventListener("mouseover", function () {
  btn.style.backgroundColor = "lightblue"; // Change color on hover
});

‚úÖ Removes events dynamically

function greet() {
  alert("This will be removed after first click!");
  btn.removeEventListener("click", greet);
}
btn.addEventListener("click", greet);

üîπ After the first click, it removes itself! (Good for one-time events).
