Type Checking & Converting Between Data Types

Overview

We use JavaScript functions like typeof, Number(), String(), and Boolean() to convert and check types. This topic also covers:

  • parseInt(), parseFloat()
  • toFixed(), toPrecision(), toLocaleString()
  • eval() (use with caution)
  • Strict mode: "use strict"

Example: Type Checking & Conversion

"use strict"; // Ensures variables are declared before use

let strNum = "42.75";
console.log(parseInt(strNum));       // 42
console.log(parseFloat(strNum));     // 42.75

let num = 123.456;
console.log(num.toFixed(2));         // "123.46"
console.log(num.toPrecision(4));     // "123.5"
console.log(num.toLocaleString());   // Output depends on locale

console.log(typeof strNum);          // "string"
console.log(typeof num);             // "number"

Practice Questions

  1. console.log(typeof NaN);        // "number"
console.log(typeof null);       // "object"
console.log(typeof undefined);  // "undefined"

    Explanation:
    • NaN (Not a Number) is still considered a number.
    • null is a special primitive value, and historically returns "object".
    • undefined means the variable was declared but not assigned a value. 

  2. let strNum = "123.456";
console.log(parseInt(strNum));            // 123 (integer)
console.log(parseFloat(strNum));          // 123.456 (float)
console.log(Number(strNum).toFixed(2));   // "123.46" (rounded)
          
  3. function isNumber(value) {
    return typeof value === "number" && !isNaN(value);
}
console.log(isNumber(42));    // true
console.log(isNumber("42"));  // false
console.log(isNumber(NaN));   // false
