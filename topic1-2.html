Best practices help write clean, maintainable, and error-free code.

1.2.2 Indentation

function greet(name) {
    console.log("Hello, " + name);
}

1.2.3 Naming Conventions

// camelCase for variables and functions
let userName = "Aman";  
function getUserInfo() { }

// UPPER_CASE for constants
const PI = 3.14159;

1.2.4 Reserved Keywords

Avoid using reserved words like var, if, return as variable names.

1.2.5 <noscript> Tag

<noscript>Your browser does not support JavaScript!</noscript>

1.2.6 Debugging with console.log and debugger

let age = 25;
console.log("Age:", age);

let x = 10;
debugger; // Pauses execution in browser
x += 5;
console.log(x);
