When a user submits a form, the browser sends data to the server.

üîπ 1. onsubmit Event: The onsubmit event runs before submission. You can validate inputs before sending. 



    <form onsubmit="return checkForm()">
      <input type="text" id="name" placeholder="Enter name">
      <input type="submit" value="Submit">
      <p id="error"></p>
    </form>
    
    <script>
      function checkForm() {
        let name = document.getElementById("name").value.trim();
        
        if (name === "") {
          document.getElementById("error").textContent = "Name cannot be blank!";
          return false; // Stops form submission
        }
        return true; // Allows form submission
      }
    </script>

üîç Explanation:

  • return false; prevents form submission if input is empty.

üîπ 2. POST vs. GET

Method Use Case Data Visibility Security
GET Retrieve data (search, filters) URL parameters (?name=John) Less secure (data in URL)
POST Send data (login, registration) Hidden in request body More secure (not visible) 
        <form action="search.php" method="GET">
          <input type="text" name="query">
          <input type="submit" value="Search">
        </form>
