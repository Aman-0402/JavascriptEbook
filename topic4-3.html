JavaScript Output Methods

In JavaScript, you can output content to an HTML document using three main methods:

  1. document.write() ‚Äì Directly writes content to the document (not recommended for dynamic updates).
  2. innerHTML ‚Äì Changes the HTML content inside an element.
  3. textContent ‚Äì Updates only the text content of an element, ignoring any HTML tags.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>JavaScript Output Methods</title>
</head>
<body>
<h1>JavaScript Output Methods</h1>
<button onclick="useDocumentWrite()">Use document.write()</button>
<button onclick="useInnerHTML()">Use innerHTML</button>
<button onclick="useTextContent()">Use textContent</button>
<div id="output"></div>
<script>
// Method 1: document.write (Overwrites the whole document if used after page load)
function useDocumentWrite() {
    document.write("<h2>This is from document.write()</h2>");
}
// Method 2: innerHTML (Modifies content inside an element)
function useInnerHTML() {
    document.getElementById("output").innerHTML = "<strong>This is from innerHTML</strong>";
}
// Method 3: textContent (Modifies text only, ignores HTML)
function useTextContent() {
    document.getElementById("output").textContent = "<strong>This is from textContent</strong>";
}
</script>
</body>
</html>

üîç Explanation:

  • document.write() ‚Äì If used after the page loads, it erases all existing content. Use it only while the page is initially loading.
  • innerHTML ‚Äì Modifies an element's HTML content, allowing HTML tags inside.
  • textContent ‚Äì Changes only the text inside an element, ignoring HTML formatting.

üõ† Debugging:

  • Issue: If you use document.write() after the page loads, it clears everything.
    Fix: Use it only in the initial script or inside window.onload.
  • Issue: textContent does not interpret HTML.
    Fix: Use innerHTML if you want to render HTML.
